将 Hugo 部署到 GitHub Pages 上去

Github Pages 本质上是一个静态网站托管系统，你可以使用它为你的每一个仓库制作一个静态网页入口。
它有两种存在方式：
a. 识别 master branch 根目录下的：README.md 或者 index.html
b. 识别 master branch 的 /docs 目录下的：README.md 或者 index.html

1. 创建一个GitHub仓库

新建一个 GitHub repository，库名为 username.github.io(username替换为你自己的用户名)

2. 构建 Hugo 静态网站

3. 将 public 文件夹转换为 Git 库

  1. 进入 public 目录，初始化 Git 库
  1. 查看 public 目录下的文件，会发现多了 .git 文件

4. 将 Git 本地库关联至远程库

  1. 在 quickstart/public 目录下，为 Git 本地库添加远程库
  1. 查看 config 文件

如果 [remote "origin"] 信息正常显示，说明你的 Git 本地库已成功关联至远程库

5. 提交你的修改至本地库

在 quickstart/public 目录下，通过 commit 提交修改，并写一个 commit message 来简洁描述你的修改

6. 将你的修改推至远程库

在 quickstart/public 目录下，将修改推至远程仓库

如无意外的话，访问https://username.github.io，你可以看到自己的博客

7. 也可以购买一个域名，绑定到github上

「传送门」：域名购买及解析教程

之后如何添加一篇博客，使用如下几步即可：

  1. 新建一篇文章，编辑内容
  2. 本地预览网站呈现效果
  3. 构建 Hugo 网站
  4. 提交修改至 Git 本地库
  5. 将修改推至远程库
